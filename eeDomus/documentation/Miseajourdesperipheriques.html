Mise à jour des périphériques

La mise à jour de l'état courant du périphérique est réalisée automatiquement:

    • Pour les widgets de type "Liste de valeurs", lorsque vous cliquez sur l'image du widget pour exécuter l'action de changement d'état du périphérique.
    • Pour les widgets de type "Valeur flottante", lorsque vous cliquez sur l'image du widget pour rafraîchir ses informations.


Mais si vous utilisez une règle vocale ou si vous passez par l'application eeDomus et que l'information change en dehors du widget, l'information n'est pas mise à jour en temps réel. Ceci peut être effectué très simplement en demandant à la box eeDomus d'envoyer l'information au plug-in lorsque l'information est modifiée afin que le widget soit mis à jour.


Nous allons prendre un exemple simple où nous voulons qu'un périphérique "Température Salon" retourne automatiquement la température de la pièce lorsque celle-ci change afin que le widget reflète tout le temps la température réelle.


Dans  votre interface eeDomus:

  1. Si cela n'est pas encore fait, créez un actionneur HTTP pour pouvoir envoyer une information au plug-in:
    1. Allez dans Accueil -> Configuration du système -> Ajouter ou supprimer un périphérique
    2. Cliquez sur "Ajouter un autre type de périphérique"
    3. Dans la section "Réseau", cliquez sur "HTTP-Actionneur"

       

    1. Entrez les informations de l'actionneur:
      • Dans l'exemple ci-dessous, le Nom, l'Usage et la Pièce sont propres à votre configuration, entrez ce que vous voulez.


    1. Cliquez sur "Sauver et continuer à éditer"
    2. Déplacez-vous dans l'onglet "Valeurs"
    3. Cliquez sur "Ajouter"
    4. Dans la colonne URL, entrez les informations suivantes:


      • http://XXX.XXX.XXX.XXX:3000/Avatar/eeDomus 


        • (où XXX.XXX.XXX.XXX est l'adresse IP de votre serveur A.V.A.T.A.R et 3000 le port utilisé)


      • Exemple:

       


    1. Sur la même ligne, cliquez dans le champ "Paramètres" pour ouvrir la fenêtre des paramètres et entrez les informations suivantes:


      •        ?command=updateWidget&periph_id=<periph_id>&client=<Client>


        • (où <periph_id> est le code API du périphérique et "<Client>" le nom d'un client A.V.A.T.A.R chez vous ! il doit exister ! )
        • Vous pouvez retrouver le code API du périphérique dans sa page d'information eeDomus ou dans les informations du périphérique dans l'onglet "Périphérique" du plug-in


      • Exemple:

       


    1. Sauvegarder les paramètres
  1. Dans la fenêtre de l'actionneur, cliquez ensuite sur "Sauvegarder"
  2. Sélectionnez ensuite le périphérique concerné:


    • Dans l'exemple ci-dessous, je sélectionne le périphérique enfant "Température" de mon détecteur:

       


  1. Dans la page de configuration du périphérique, cliquez sur l'onglet "Règles associées"
  2. Cliquez sur "Nouvelle règle" et entrez la règle suivante:
    • Critère: <Périphérique température> change de valeur
    • Action: <Actionneur HTTP> exécute <Etat 1>


       


  1. Cliquez sur "Sauver"
  2. Terminé !!!
  3. Lorsque la température changera, le widget "Température" A.V.A.T.A.R se mettra à jour automatiquement.


Nous pouvons maintenant créer de nouveau états dans l'actionneur HTTP dédié à cette pièce (Salon) pour envoyer d'autres informations, comme par exemple, si une lumière s'allume ou s’éteint, pour un détecteur de mouvement...  bref.... tout ce qu'on veut afin que les widgets reflètent toutes les informations en temps réel.
